## Workflow 1 ‚Äî `/onboard <client name>`

**Effect:** Cuts client onboarding from 2 hours ‚Üí 5 minutes. Spawns a private client channel with starter canvas + DMs you the WhatsApp invite link to send.

**Trigger:**
- Type: **Shortcut from a channel** (or "From a link" for slash-command-style)
- Name: `Onboard new client`
- Description: `Spawn private #client-<name> channel with onboarding canvas`
- Run from: any channel (or restrict to `#daily-ops`)

**Form input:**
| Variable | Label | Type | Required |
|---|---|---|---|
| `client_name` | Client name (e.g. "Flamingo") | Single-line text | Yes |
| `service_pack` | Service pack | Dropdown (Pack Crecimiento, Pack Dominaci√≥n, Custom, Trojan Free) | Yes |
| `monthly_mxn` | Monthly fee (MXN) | Number | Yes |
| `start_date` | Service start date | Date | Yes |
| `primary_email` | Client primary email | Email | Yes |

**Steps:**

1. **Send a message** to channel: (use a variable to dynamically pick `#client-{{client_name | lowercase | replace_spaces_with_dashes}}` ‚Äî but Workflow Builder can't create channels itself, so this step actually sends to a fallback channel)
   - Alternative: send to `#daily-ops` saying "New client onboarding triggered: {{client_name}}. Create channel #client-{{client_name | lowercase}} now via the + icon."

2. **Send a DM to you** with this message:
```
üéâ New client kicked off ‚Äî {{client_name}}

Pack: {{service_pack}}
Monthly: ${{monthly_mxn}} MXN
Start: {{start_date}}
Email: {{primary_email}}

‚ö° Next steps (in order):
1. Create channel #client-{{client_name | lowercase}} in Slack
2. Send WhatsApp welcome (template below)
3. Schedule kickoff call via Calendly
4. Run /quote {{client_name}} to generate the cotizaci√≥n
5. Post the verified KPIs canvas in their channel

üì± WhatsApp template (copy-paste):
"Hola {{client_name}}, bienvenidos a JegoDigital. Aqu√≠ va el link a su Calendly para que agendemos la llamada de kickoff: https://calendly.com/jegoalexdigital/30min ¬°Hablamos pronto!"
```

3. **Save to Firestore** (optional, requires webhook step):
   - URL: `https://us-central1-jegodigital-e02fb.cloudfunctions.net/onboardClient`
   - Method: POST
   - Body: all variables as JSON

**Backend Cloud Function needed:** `onboardClient` (see code in section 5 below)

## Workflow 4 ‚Äî `/end-day` close-of-day ritual

**Effect:** Forces a clean shutdown of the day with 3 questions. No more "did I do today's big rock?" guilt.

**Trigger:**
- Type: **Scheduled** (run every weekday 8:00 PM Cancun)
- Or: **Shortcut from a channel** for manual trigger

**Form input:** None at trigger (form opens during workflow)

**Steps:**

1. **Send a DM to you** with form attached:
```
üåô End-of-day check-in ‚Äî {{date}}

Take 60 seconds. Tomorrow you'll thank yourself.
```

2. **Open form** with these questions:
   - Q1 (Yes/No): Did you ship today's Big Rock?
   - Q2 (Single-line text): If yes, what proof exists? (Slack link, commit hash, post URL)
   - Q3 (Single-line text): What's tomorrow's Big Rock?

3. **Send a message** to `#daily-ops`:
```
‚úÖ End-of-day ‚Äî {{date}}

Big Rock done: {{q1_answer}}
Proof: {{q2_answer}}
Tomorrow's rock: {{q3_answer}}

Closed at {{workflow_started_at}}.
```

4. **Save to Firestore** (optional webhook):
   - URL: `https://us-central1-jegodigital-e02fb.cloudfunctions.net/saveEndOfDay`
   - Body: `{ date, big_rock_done, proof, tomorrow_rock }`

**Backend:** Optional ‚Äî pure Workflow Builder works for the Slack-only version.

## 5. Cloud Function backends

### 5a. `onboardClient` (skeleton)

```javascript
// website/functions/slackWorkflows.js
const functions = require("firebase-functions");
const admin = require("firebase-admin");
const axios = require("axios");
const { slackPost } = require('./slackPost');

if (!admin.apps.length) admin.initializeApp();

exports.onboardClient = functions.https.onRequest(async (req, res) => {
    const { client_name, service_pack, monthly_mxn, start_date, primary_email } = req.body;

    // 1. Save to Firestore
    const docRef = await admin.firestore().collection("clients").add({
        client_name,
        service_pack,
        monthly_mxn: Number(monthly_mxn),
        start_date,
        primary_email,
        status: "onboarding",
        created_at: admin.firestore.FieldValue.serverTimestamp(),
    });

    // 2. Trigger client-onboarding skill via webhook OR enqueue a task
    // (full skill implementation in skills/client-onboarding)

    // 3. Post confirmation to #revenue
    await slackPost('revenue', {
        text: `üÜï New client onboarded ‚Äî ${client_name}`,
        blocks: [
            { type: "header", text: { type: "plain_text", text: `üÜï ${client_name} onboarded` } },
            { type: "section", text: { type: "mrkdwn", text: `Pack: *${service_pack}*\nMRR: $${monthly_mxn} MXN\nStart: ${start_date}` } },
        ],
    });

    res.status(200).json({ ok: true, client_id: docRef.id });
});
```

### 5c. `saveEndOfDay` (skeleton)

```javascript
exports.saveEndOfDay = functions.https.onRequest(async (req, res) => {
    const { date, big_rock_done, proof, tomorrow_rock } = req.body;
    await admin.firestore().collection("end_of_day").doc(date).set({
        big_rock_done,
        proof,
        tomorrow_rock,
        saved_at: admin.firestore.FieldValue.serverTimestamp(),
    });
    res.status(200).json({ ok: true });
});
```
