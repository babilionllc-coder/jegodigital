**Good news:** All 3 agents have all the booking/audit tools they need. No missing tools. (Earlier draft of this audit claimed missing tools — retracted.)

**Bad news:** `silence_end_call_timeout = None` on all 3. Default is -1 (never end on silence). Combined with `initial_wait_time = None` (default 4s), this means:
- Call connects → dead air → ElevenLabs NEVER auto-ends → Twilio hangs up eventually → ElevenLabs leaves conversation as `status=initiated` or `failed` forever.
- **This is why 25/30 of today's conversations never reached `status=done`** and no post-call webhook fired.

**Action CC-8 (P0 — replaces old):** PATCH all 3 agents to `silence_end_call_timeout=20`, `initial_wait_time=1.0`. Caveat: first PATCH attempt today returned 200 but silently dropped the fields (per memory `elevenlabs_silence_timeout_fix.md`). Try PUT with full config shape, or test via dashboard + diff. See task #9.

### The real cold-call failure chain (corrected diagnosis)

1. `coldCallRun` dials 50 leads via Twilio ✅ (works)
2. ElevenLabs mints `conversation_id` and seeds `call_analysis.outcome=pending` ✅
3. Call connects on a phone that goes to voicemail or the person answers and says nothing
4. `silence_end_call_timeout=None` → ElevenLabs never terminates the session
5. Twilio eventually hangs up, leaving ElevenLabs conversation in `initiated`/`failed` state
6. **`status=done` is never reached → post-call webhook `elevenLabsWebhook` never fires**
7. `call_analysis.outcome` stays "pending" forever → coldCallReport shows `connected=0, pending=61`

**Fix = set silence_end_call_timeout. The webhook itself is not broken.** Only 5 of 30 calls today reached `done`; only 1 had real conversation messages. That's the actual problem to solve.
