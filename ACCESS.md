## 🚨 THE RULE (read this ONCE, never forget)

```
┌────────────────────────┐    ┌──────────────────────┐    ┌──────────────────┐
│ GitHub Secrets (37)    │──▶│ deploy.yml heredoc    │──▶│ functions/.env    │
│ Source of truth        │    │ Must name each key    │    │ Live Cloud Fns    │
│ github.com/.../        │    │ .github/workflows/    │    │ (auto-generated   │
│ settings/secrets       │    │ deploy.yml            │    │ on every deploy)  │
└────────────────────────┘    └──────────────────────┘    └──────────────────┘
```

**A key reaches production ONLY when:** (a) in GitHub Secrets AND (b) its name is in the heredoc in `.github/workflows/deploy.yml`. Missing either = stranded.

**Keys in GH but NOT in heredoc = BACKUP ONLY** (used from local dev or GH Actions directly, not injected into Cloud Functions runtime). This is valid — not every key needs to reach production.

**WHY WE HAD STRANDED KEYS:** The heredoc is a hardcoded whitelist. Every new key needs two adds (Secrets + heredoc). The Apr 20 audit caught 6 stranded secrets and added them.

**THE GUARDRAIL:** This file + the `envAudit` cron that posts green ✅ or red 🚨 to Slack at 06:00 UTC daily.

## ✅ HOW TO ADD A NEW KEY

1. **Add to GitHub Secrets:** https://github.com/babilionllc-coder/jegodigital/settings/secrets/actions
2. **If it needs to reach Cloud Functions:** add line to heredoc in `.github/workflows/deploy.yml`:
   ```yaml
   KEY_NAME=${{ secrets.KEY_NAME }}
   ```
3. **Add row to this file's registry** with purpose + who uses it
4. **If production-critical:** add to `envAudit.js` REQUIRED_KEYS list
5. **Push any tiny change to `website/**`** (or the workflow file) → deploy fires → new `functions/.env` has the new value
6. **Verify:** next day's 6am Slack digest via `envAudit` reports "all keys present ✅"

## 🔄 HOW TO ROTATE A KEY (when you actually want to)

1. Generate new key at provider (Brevo, Instantly, etc.)
2. Update GitHub Secret (same name, new value) via UI or REST API
3. Bump a commit (e.g., touch any workflow file) → deploy fires → new `functions/.env` has new value
4. Deprecate the old key at the provider
5. Update this file's changelog
