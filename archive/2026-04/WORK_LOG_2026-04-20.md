## 1. Fixed jego_hero_bg.webm 404 on jegodigital.com

**Problem.** Homepage hero video returned 404 from jegodigital.com. Root cause: `firebase.json` `hosting.ignore` contained a bare `videos/**` glob, which Firebase interprets relative to the `public` directory — so it was matching `website/videos/**` and blocking the hero webm from ever being deployed. A stale `!videos/*.webm` negation was in place but Firebase negation semantics are unreliable and it wasn't covering the case.

**Fix.** Removed both `videos/**` and `!videos/*.webm` from `hosting.ignore`. Final ignore list is now tightly scoped to directories that genuinely shouldn't ship (`node_modules`, `tiktok vids`, `tools`, `functions`).

**CI hardening.** `.github/workflows/validate-videos.yml` was updated to treat the bare glob forms as forbidden patterns so the bug can't silently return. Forbidden list now includes: `videos/**`, `website/videos/**`, `assets/video/**`, `website/assets/video/**`, `assets/videos/**`, `website/assets/videos/**`, `cf-videos/**`, `website/cf-videos/**`, `img/showcase/**`, `website/img/showcase/**`, and the `!videos/*.webm` negation.

**Commit.** `d72d02c — fix hosting drop videos glob from firebase.json and harden CI guard`

**Verification.** `curl -sI https://jegodigital.com/videos/jego_hero_bg.webm` → `HTTP/2 200`. GH Actions deploy workflow green.

## 3. Git plumbing: squash-onto-tip recovery technique

The favicon commit was built on `d72d02c`, but origin had advanced to `fd3c7ad` (cold-call autopilot commit) with a separate file set. A normal `git pull --rebase` kept failing because the working tree had residual unstaged changes in `website/functions/` that couldn't be cleanly handled.

Used the plumbing recipe from `jegodigital_push_protections.md` to build a clean fast-forward commit directly:

1. Copied `.git/index` to `/tmp/altindex-favicon`
2. `GIT_INDEX_FILE=/tmp/altindex-favicon git read-tree fd3c7ad` — loaded origin's tree
3. For each file in `git diff --name-status d72d02c 3454156`, ran `update-index --add --cacheinfo` with the blob SHA from the favicon commit's tree
4. `write-tree` → new tree `99965c3c1305b3bf9a77f3a2782b71c7e738972d`
5. `commit-tree` with `-p fd3c7ad` → new commit `e8e9f00`
6. Overwrote `.git/refs/heads/main` directly (sandbox can't `unlink` but CAN overwrite)

Result: a linear fast-forward push, no rebase needed, no working-tree touched, residual unstaged changes in `website/functions/` left alone for Alex to resolve on his own time.

**Pre-push checks run and passed:** no file >50MB, no API-key strings, no skill docs.
